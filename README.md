# 🍽 Eat&fit — Telegram-бот службы доставки еды

Полноценный Telegram-бот для службы доставки **Eat&fit** на двух языках (русский / узбекский).
Стек: **Node.js**, **node-telegram-bot-api**, **PostgreSQL**.

## Возможности

- 🌐 Двуязычный интерфейс (ru/uz), язык выбирается при первом входе
- 📝 Регистрация клиента (имя, фамилия, телефон, адрес)
- 🍽 Меню по категориям (основные блюда, напитки, десерты)
- 🔗 Система заказа через **deep link** (`?start=item_<id>`)
- 🛒 Корзина в памяти, оформление с выбором оплаты (баланс / при получении)
- 💰 Баланс и история пополнений (депозиты)
- 👨‍🍳 Роль повара: принять в работу → готово
- 🚗 Роль курьера: взять доставку → доставил
- 🔧 Админ-панель: меню, пользователи, депозиты, заказы, генерация постов для канала
- 🔔 Уведомления в группы поваров/курьеров и клиенту о смене статуса
- 💳 Атомарное создание заказа в транзакции PostgreSQL

## 📦 Установка

### 1. Клонирование и зависимости

```bash
git clone <repo-url>
cd eatfit-bot
npm install
```

### 2. Настройка окружения

Скопируйте `.env.example` в `.env` и заполните значения:

```bash
cp .env.example .env
```

### 3. Создание базы данных

Создайте БД в PostgreSQL и примените схему:

```bash
# создать БД (пример)
createdb eatfit

# применить схему
npm run db:init
# либо вручную:
# psql "$DATABASE_URL" -f db/schema.sql
```

### 4. Запуск

```bash
npm start
```

Бот запустится в режиме polling (`polling: true`).

## ⚙️ Переменные окружения (`.env`)

| Переменная            | Описание                                                                 |
|-----------------------|--------------------------------------------------------------------------|
| `BOT_TOKEN`           | Токен бота от [@BotFather](https://t.me/BotFather)                        |
| `DATABASE_URL`        | Строка подключения к PostgreSQL (`postgresql://user:pass@host:5432/db`)   |
| `ADMIN_TELEGRAM_IDS`  | Telegram ID администраторов через запятую. Им автоматически выдаётся роль `admin` |
| `COOK_GROUP_ID`       | ID группы поваров (куда падают новые заказы). Формат `-100xxxxxxxxx`      |
| `COURIER_GROUP_ID`    | ID группы курьеров (куда падают готовые заказы)                           |
| `BOT_USERNAME`        | Username бота **без `@`** — используется для генерации deep link          |

> Чтобы получить ID группы, добавьте бота в группу и временно залогируйте `msg.chat.id`,
> либо используйте сервисных ботов вроде `@getmyid_bot`.

## 👑 Как добавить первого администратора

Два способа:

**Способ 1 (рекомендуется).** Добавьте свой Telegram ID в `ADMIN_TELEGRAM_IDS` в `.env`.
При `/start` бот автоматически проставит роль `admin` в БД.

**Способ 2.** Прямой UPDATE в БД (пользователь должен сначала запустить `/start`):

```sql
UPDATE clients SET role = 'admin' WHERE telegram_id = 123456789;
```

Аналогично можно назначить повара или курьера:

```sql
UPDATE clients SET role = 'cook'    WHERE telegram_id = 111111111;
UPDATE clients SET role = 'courier' WHERE telegram_id = 222222222;
```

Роли также меняются из админ-панели: **👥 Пользователи → поиск → 🔁 Изменить роль**.

## 🔗 Система Deep Link

Каждое блюдо имеет ссылку вида:

```
https://t.me/<BOT_USERNAME>?start=item_<menu_item_id>
```

**Как это работает:**

1. Админ создаёт блюдо в панели — бот сразу выдаёт deep link и готовый пост для канала.
2. Пост публикуется в Telegram-канале:

   ```
   🍕 Лагман
   📝 Классический узбекский лагман
   💵 25 000 сум

   👉 Заказать: https://t.me/eatfit_bot?start=item_42
   ```

3. Клиент нажимает на ссылку → бот открывается, парсит `start=item_42` в `/start` handler.
4. Сразу показывается **карточка блюда** с кнопками `➕ Добавить в корзину` / `❌ Не надо`.
5. После добавления — предложение добрать **напитки 🥤** и **десерты 🍰**, затем `✅ Оформить заказ`.
6. Подтверждение корзины → выбор оплаты (с баланса или при получении) → заказ создаётся.

Генерация ссылок и постов — в `src/utils/deeplink.js`.
Массовая генерация постов: **Админ-панель → 📢 Генерировать пост → 📢 Все активные блюда**.

## 🔄 Жизненный цикл заказа

| Статус       | Что происходит                                | Кто меняет |
|--------------|-----------------------------------------------|------------|
| `pending`    | Заказ создан, ушёл в группу поваров            | система    |
| `cooking`    | «Принять в работу» — уведомление клиенту       | повар      |
| `delivering` | «Готово» — уходит в группу курьеров            | повар      |
| `done`       | «Доставил» — уведомление клиенту               | курьер     |
| `cancelled`  | Отмена                                         | —          |

При каждой смене статуса клиент получает уведомление (на своём языке).
