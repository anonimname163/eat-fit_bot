# Claude Code Prompt — Eat&fit Telegram Bot

## Контекст проекта

Создай полноценного Telegram-бота для службы доставки еды **Eat&fit**.
Стек: **Node.js**, библиотека **node-telegram-bot-api**, база данных **PostgreSQL**.
Интерфейс бота на двух языках: **русский и узбекский** (пользователь выбирает язык при первом входе).

---

## Структура базы данных

Создай файл `db/schema.sql` со следующими таблицами:

```sql
-- Клиенты
clients (
  id SERIAL PRIMARY KEY,
  telegram_id BIGINT UNIQUE NOT NULL,
  first_name VARCHAR(100),
  last_name VARCHAR(100),
  phone VARCHAR(20),
  address TEXT,
  language VARCHAR(5) DEFAULT 'ru',  -- 'ru' или 'uz'
  balance NUMERIC(10,2) DEFAULT 0,   -- депозит/предоплата
  role VARCHAR(20) DEFAULT 'client', -- 'client' | 'admin' | 'cook' | 'courier'
  created_at TIMESTAMP DEFAULT NOW()
)

-- Меню
menu_items (
  id SERIAL PRIMARY KEY,
  name_ru VARCHAR(200),
  name_uz VARCHAR(200),
  description_ru TEXT,
  description_uz TEXT,
  category VARCHAR(50),  -- 'main' | 'drink' | 'dessert'
  price NUMERIC(10,2),
  photo_url TEXT,
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT NOW()
)

-- Заказы
orders (
  id SERIAL PRIMARY KEY,
  client_id INTEGER REFERENCES clients(id),
  status VARCHAR(30) DEFAULT 'pending',
  -- статусы: pending | confirmed | cooking | delivering | done | cancelled
  total_amount NUMERIC(10,2),
  paid_from_balance NUMERIC(10,2) DEFAULT 0,
  delivery_address TEXT,
  comment TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
)

-- Позиции заказа
order_items (
  id SERIAL PRIMARY KEY,
  order_id INTEGER REFERENCES orders(id),
  menu_item_id INTEGER REFERENCES menu_items(id),
  quantity INTEGER,
  price_at_order NUMERIC(10,2)
)

-- Депозиты (история пополнений)
deposits (
  id SERIAL PRIMARY KEY,
  client_id INTEGER REFERENCES clients(id),
  amount NUMERIC(10,2),
  note TEXT,
  created_by BIGINT,  -- telegram_id админа который пополнил
  created_at TIMESTAMP DEFAULT NOW()
)
```

## Детальное описание функционала

### 1. Первый запуск — регистрация (`/start`)

Если пользователь новый (нет в таблице `clients`):
1. Предложить выбор языка: кнопки `🇷🇺 Русский` / `🇺🇿 O'zbek`
2. Попросить **имя** (текстом)
3. Попросить **фамилию** (текстом)
4. Попросить **номер телефона** — кнопка «Поделиться контактом» + ввод вручную
5. Попросить **адрес доставки** (текстом)
6. Сохранить в `clients`, показать главное меню

Если пользователь уже зарегистрирован — сразу главное меню.

### 2. Главное меню клиента

Inline или Reply кнопки:
- 🍽 Сделать заказ / Buyurtma berish
- 👤 Мой профиль / Profilim
- 💰 Мой баланс / Balansim
- 📦 Мои заказы / Buyurtmalarim

### 3. Система заказа через Deep Link

**Генерация ссылки:**
- Каждое блюдо имеет ссылку вида: `https://t.me/BOT_USERNAME?start=item_<menu_item_id>`
- Утилита `deeplink.js` генерирует эти ссылки для всех блюд

**Пост для Telegram-канала (генерируется автоматически через команду админа):**
```
🍕 Название блюда
📝 Описание
💵 Цена: 25 000 сум

👉 Заказать: https://t.me/BOT_USERNAME?start=item_42
```

**Когда клиент переходит по ссылке:**
1. Бот открывается с уже выбранным блюдом
2. Показывает карточку блюда с кнопками: `➕ Добавить в корзину` / `❌ Не надо`
3. После добавления: «Добавить что-то ещё?» с категориями **Напитки 🥤** и **Десерты 🍰**
4. Показывает список напитков / десертов с кнопками добавления
5. Кнопка `✅ Оформить заказ`
6. Подтверждение: показывает корзину с итогом, адрес доставки, баланс
7. Если баланса достаточно — предложить списать с баланса
8. Подтверждение → заказ создаётся в БД

**Обычный заказ (без deeplink):**
- Клиент видит меню по категориям, набирает корзину сам

### 4. Уведомления после оформления заказа

**В группу поваров** (`COOK_GROUP_ID` из `.env`):
```
🆕 Новый заказ #42
─────────────────
🍕 Лагман х2
🥤 Чай зелёный х1
🍰 Тирамису х1
─────────────────
📝 Комментарий: без лука
⏰ 14:35
```
Кнопка: `✅ Принять в работу`

**В группу курьеров** (`COURIER_GROUP_ID` из `.env`):
```
📦 Заказ #42 готов к доставке
📍 Адрес: ул. Навои 15, кв. 7
📞 Телефон: +998 90 123 45 67
⏰ 14:52
```
Кнопка: `🚗 Взять доставку`

### 5. Статусы заказа и уведомления клиенту

При каждом изменении статуса бот отправляет клиенту сообщение:
- `confirmed` → «Ваш заказ принят, готовим!»
- `cooking` → «Повар приступил к готовке 👨‍🍳»
- `delivering` → «Курьер выехал! 🚗»
- `done` → «Заказ доставлен. Приятного аппетита! 🙏»
- `cancelled` → «Заказ отменён»

### 6. Роли и их интерфейсы

#### 👨‍🍳 Повар (`role = 'cook'`)
При `/start` видит кнопку «Режим повара».
В режиме повара:
- Видит входящие заказы (список активных)
- Кнопка «Принять в работу» → статус меняется на `cooking`, уведомление клиенту
- Кнопка «Готово» → статус `delivering`, уведомление в группу курьеров

#### 🚗 Курьер (`role = 'courier'`)
При `/start` видит кнопку «Режим курьера».
В режиме курьера:
- Видит заказы со статусом `delivering`
- Кнопка «Взять доставку» → закрепляет заказ за курьером
- Кнопка «Доставил» → статус `done`, уведомление клиенту

#### 🔧 Администратор (`role = 'admin'`)
Команда `/admin` открывает панель:

```
⚙️ Панель администратора
────────────────────────
📋 Управление меню
👥 Пользователи
💰 Депозиты
📊 Заказы
📢 Генерировать пост для канала
```

**Управление меню:**
- Список всех блюд с кнопками «Редактировать» / «Скрыть» / «Удалить»
- «Добавить блюдо» — пошаговый диалог: название RU → название UZ → описание RU → описание UZ → категория → цена → фото (опционально)
- После создания блюда — сразу генерировать deep link и пост для канала

**Пользователи:**
- Поиск по номеру телефона или Telegram username
- Просмотр профиля: имя, баланс, история заказов
- Изменение роли (client/cook/courier/admin)

**Депозиты:**
- Пополнить баланс клиента: ввести telegram_id или телефон → сумма → подтверждение
- История пополнений

**Генерация поста для канала:**
- Выбрать блюдо из меню → бот выдаёт готовый текст поста с deep link
- Отдельно: сгенерировать посты для всех активных блюд сразу (batch)

### 7. Конфигурация `.env`

```env

DATABASE_URL=postgresql://user:password@localhost:5432/eatfit
ADMIN_TELEGRAM_IDS=123456789,987654321   # через запятую
COOK_GROUP_ID=-100xxxxxxxxx
COURIER_GROUP_ID=-100xxxxxxxxx
BOT_USERNAME=eatfit_bot                  # без @, для deep links
```

## Требования к коду

1. **Обработка ошибок** — все async функции в try/catch, пользователь видит понятное сообщение при ошибке
2. **Защита от спама** — не обрабатывать дублирующиеся callback_query
3. **Транзакции PostgreSQL** — при создании заказа использовать транзакцию (списание баланса + создание order + order_items атомарно)
4. **Middleware авторизации** — перед каждым handler проверять роль
5. **i18n** — все тексты вынести в `src/i18n.js`, никакого хардкода строк в handlers
6. **Логирование** — использовать `console.log` с префиксами `[INFO]`, `[ERROR]`, `[ORDER]`
7. **Корзина** — хранить в памяти (Map по telegram_id) пока заказ не оформлен
8. **Команды бота** (`setMyCommands` при старте):
   - `/start` — главное меню
   - `/menu` — меню ресторана
   - `/orders` — мои заказы
   - `/profile` — мой профиль
   - `/admin` — (только для admins)

## Важные детали реализации

- `node-telegram-bot-api` использовать с `polling: true` для разработки
- Для callback кнопок использовать паттерн `action:payload`, например `order:confirm:42`
- Deep link параметр парсить в `/start` handler: если `msg.text` содержит `start=item_<id>` — сразу открыть карточку блюда
- Генерация deep link: `https://t.me/${process.env.BOT_USERNAME}?start=item_${menuItemId}`
- При пополнении баланса — запись в таблицу `deposits` + UPDATE `clients.balance`
- Фото блюд — принимать как `file_id` от Telegram и хранить в `menu_items.photo_url`
